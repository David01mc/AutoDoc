Cómo funciona

Tres pasos. Cero interrupciones.

AutoDoc se instala una vez y trabaja en silencio. Tú trabajas con Claude Code exactamente igual que siempre.

1

Instalas AutoDoc

Ejecuta install.sh o install.bat apuntando a tu proyecto. En 60 segundos está listo.

2

Trabajas como siempre

Abre Claude Code y trabaja exactamente igual. AutoDoc opera en el fondo, invisible. No cambia nada de tu flujo.

3

El diario se escribe solo

Tras cada respuesta de Claude, una entrada estructurada se añade a docs/YYYY-MM-DD.md. Automáticamente.

Sin llamadas a la API. Sin tokens. Completamente gratis.

AutoDoc lee la transcripción local del hook Stop de Claude Code. No hace ninguna petición a Anthropic. Cada entrada del diario cuesta exactamente 0 tokens adicionales.

Configuración

Un JSON. Todo el control.

Edita Claude_Scripts/autodoc.config.json para adaptar AutoDoc a tu proyecto.

autodoc.config.json
1{
2 "lang": "es",
3 "docs_dir": "docs",
4 "max_summary_lines": 3,
5 "max_changes_in_summary": 15,
6 "min_message_length": 30,
7 "skip_read_only_turns": true
8}
"lang"

"es" o "en" — idioma del diario generado. Títulos, verbos y mensajes se adaptan automáticamente.

"docs_dir"

Carpeta donde se escriben los diarios. Puede ser cualquier ruta relativa a la raíz del proyecto.

"max_summary_lines"

Líneas extraídas de la respuesta de Claude para el campo Resultado. Ajusta según tus preferencias de verbosidad.

"min_message_length"

Mínimo de caracteres en el mensaje del usuario para activar el log. Evita registrar turnos triviales.

"skip_read_only_turns"

true omite turnos donde Claude solo leyó archivos sin modificar nada. Mantiene el diario limpio.

En marcha en 60 segundos.

Instalación manual (paso a paso)

  1. Copia Claude_Scripts/ y .claude/commands/ en la raíz de tu proyecto.
  2. Crea o edita .claude/settings.local.json con el siguiente hook:
.claude/settings.local.json
1{
2 "hooks": {
3 "Stop": [
4 { "hooks": [{
5 "type": "command",
6 "command": "python Claude_Scripts/log_activity.py",
7 "timeout": 15
8 }]}]
9 }}

Listo. Abre el proyecto en Claude Code y empieza a trabajar.
